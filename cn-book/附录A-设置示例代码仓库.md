# 附录 A 设置示例代码仓库

本书的配套仓库托管了各章节附带的示例代码。它演示了如何使用 OpenAI 的工具和 API 构建并运行一个 AI 智能体，并且每个章节的示例都组织在各自独立的文件夹中。在你能够运行书中的代码清单之前，你需要在本地机器上有一份可用的仓库副本、一个符合示例要求的 Python 环境，以及一个通过本地环境文件连接的有效 OpenAI API 密钥。

本附录将从头到尾指导你完成这个设置过程。它分为四个部分，遵循自然的设置顺序：克隆仓库、创建 Python 环境、安装依赖项并配置 API 密钥，最后运行一个示例智能体以确认一切连接正确。第一次操作时请按顺序阅读各部分；后续操作时，你可以直接跳转到所需的步骤。

## A.1 克隆仓库

首先，将仓库克隆到你的本地机器上。该仓库托管在 GitHub 上，可以通过标准的 `git clone` 命令获取。打开一个终端，切换到存放项目的目录，然后运行以下命令：

git clone https://github.com/cxbxmxcx/AI-Agent-Workflows.git
cd AI-Agent-Workflows

第一条命令会将仓库的完整副本下载到一个名为 AI-Agent-Workflows 的新文件夹中。第二条命令将你移入该文件夹，以便后续的设置步骤在正确的位置运行。

## A.2 创建 Python 环境

该项目需要 Python 3.11 或更高版本。推荐的做法是为该仓库创建一个专用的虚拟环境，这样它的依赖项就不会与你机器上的其他 Python 项目冲突。具体的命令取决于你的操作系统，因此以下部分将分别为 Windows 和 macOS 或 Linux 提供说明。

在 Windows 上，在仓库根目录的命令提示符或 PowerShell 窗口中运行：

python -m venv venv
venv\Scripts\activate

在 macOS 或 Linux 上，在仓库根目录的终端中运行：

python3 -m venv venv
source venv/bin/activate

无论在哪个平台上，一旦环境被激活，你的 shell 提示符前会出现 `(venv)` 前缀，这确认了后续的 `python` 和 `pip` 命令将在虚拟环境内运行。

如果你更倾向于使用现有的 Python 安装，而不是之前创建的虚拟环境，你可以将 VS Code 指向该解释器。为此，请按照以下步骤操作：

1. 在 Windows 或 Linux 上按 Ctrl-Shift-P，或在 macOS 上按 Cmd-Shift-P，打开命令面板。
2. 搜索 **Python: Select Interpreter**。
3. 选择你想为该项目使用的 Python 解释器。

## A.3 安装依赖项并配置环境

有了一个可用的 Python 环境后，下一步是安装示例所依赖的包，并让它们能够访问你的 OpenAI API 密钥。该仓库在其根目录下的 requirements.txt 文件中固定了依赖项，并从本地的 .env 文件中读取你的 API 密钥。本节将依次介绍两种安装路径和 API 密钥配置。

### A.3.1 路径 A：使用 VS Code 调试器

如果你安装了带有 Python 扩展的 VS Code，可以按 F5 键开始调试某个示例文件。仓库附带的启动配置确保在示例运行之前自动安装所需的依赖项。如果你不想手动管理依赖项，这是最快的路径。

### A.3.2 路径 B：使用 pip 手动安装

如果你更喜欢自己安装依赖项，或者你是在普通终端中运行示例，请在虚拟环境激活的情况下，在仓库根目录运行以下命令：

pip install -r requirements.txt

`pip` 会下载每个固定的包并将其安装到激活的虚拟环境中。根据你的网络连接情况，第一次安装可能需要一两分钟。

### A.3.3 配置 OpenAI API 密钥

这些示例通过从仓库根目录的 .env 文件中读取的 API 密钥来认证 OpenAI。该仓库包含一个 .env.example 文件，你可以将其复制作为起点。在仓库根目录创建一个名为 .env 的新文件，包含以下内容：

OPENAI_API_KEY=your_openai_api_key_here

将 `your_openai_api_key_here` 替换为你实际的 OpenAI API 密钥。你可以从 <https://platform.openai.com/account/api-keys> 的 OpenAI API Keys 页面获取密钥。

> 警告：切勿分享或将你的 .env 文件提交到版本控制系统中。它包含一个可以访问你 OpenAI 账户的秘密凭证，任何获得它的人都可能给你的账户带来费用。

## A.4 运行示例代码

完成克隆仓库、激活虚拟环境、安装依赖项并配置 API 密钥后，你就可以运行其中一个示例了。本节将向你展示如何运行一个示例、处理你可能遇到的最常见问题，并养成一些小的习惯，这些习惯将使你的设置在本书学习过程中保持健康。

### A.4.1 运行一个示例

每章的代码都位于仓库根目录下自己的文件夹中。要运行一个示例，请导航到该章文件夹并执行 Python 文件。例如，要运行第 2 章的第一个智能体：

python chapter_02/01_first_agent.py

该智能体将运行至完成，并将其输出打印到终端。你可以用相同模式运行仓库中的任何其他示例，只需将路径替换为你想要运行的章节和文件即可。

### A.4.2 排查常见问题

大多数设置问题归结为两个根本原因之一：API 密钥缺失或不正确，或激活的 Python 解释器不对。以下两种模式几乎涵盖了所有你可能遇到的问题：

* 如果你看到身份验证错误，请再次检查你的 .env 文件是否在仓库根目录，并且 `OPENAI_API_KEY` 是否设置为有效的密钥。
* 如果你看到 `ModuleNotFoundError`，请确认你的虚拟环境已激活，并且依赖项是安装在该环境中的。

### A.4.3 保持你的设置健康

几个小习惯将在你阅读本书剩余部分时为你节省时间：

* 确保你使用的 Python 解释器确实与你设置的环境匹配。将系统 Python 与项目虚拟环境混合使用是导致导入错误的最常见原因。
* .env 文件列在 .gitignore 中，绝不应提交到版本控制系统。将其保留在你的本地机器上，以确保你的 API 密钥安全。
* 如果你因任何原因轮换了你的 OpenAI API 密钥，请在再次运行示例之前更新 .env 中的值。
